# Vikingship 项目入门指南

## 📚 项目概览

这是一个**React + TypeScript 组件库**教学项目，包含多个 UI 组件（Button、Input、Form、Menu 等）。

## 🚀 快速开始

### 第一步：安装依赖

```bash
cd vikingship
npm install
# 或
yarn install
```

### 第二步：运行项目（推荐方式）

这个项目主要通过 **Storybook** 来展示和开发组件，这是查看所有组件效果的最佳方式：

```bash
npm run storybook
```

运行后会自动打开浏览器，访问 `http://localhost:6006`，你可以看到：
- 所有组件的可视化展示
- 组件的不同状态和用法
- 交互式文档

### 其他运行方式

如果你想用传统的 React 开发模式：

```bash
npm start
```

这会启动 React 开发服务器（但需要先创建 App.tsx 入口文件）。

## 📖 学习路径建议

### 1. 从最简单的组件开始

**推荐顺序：**
1. **Button** (`src/components/Button/`) - 最简单的组件
2. **Icon** (`src/components/Icon/`) - 图标组件
3. **Input** (`src/components/Input/`) - 输入框
4. **Alert** (`src/components/Alert/`) - 提示框
5. **Menu** (`src/components/Menu/`) - 菜单（较复杂）
6. **Form** (`src/components/Form/`) - 表单（最复杂）

### 2. 每个组件的文件结构

每个组件通常包含：
- `组件名.tsx` - 组件的主要实现代码
- `index.tsx` - 导出文件
- `_style.scss` - 样式文件
- `组件名.stories.tsx` - Storybook 故事文件（展示组件用法）
- `组件名.test.tsx` - 单元测试文件

### 3. 阅读顺序（以 Button 为例）

1. **先看 `button.stories.tsx`** - 了解组件的用法和效果
2. **再看 `button.tsx`** - 理解组件的实现逻辑
3. **查看 `_style.scss`** - 了解样式实现
4. **最后看 `button.test.tsx`** - 学习如何测试组件

## 🔍 关键文件说明

### 项目入口
- `src/index.tsx` - 导出所有组件，这是组件库的主入口

### 样式系统
- `src/styles/index.scss` - 全局样式入口
- `src/styles/_variables.scss` - 样式变量（颜色、尺寸等）
- `src/styles/_mixin.scss` - SCSS 混入（可复用的样式函数）

### 工具函数
- `src/hooks/` - 自定义 React Hooks
  - `useClickOutside.tsx` - 点击外部区域检测
  - `useDebounce.tsx` - 防抖函数

## 💡 学习建议

1. **先运行 Storybook**，在浏览器中看到所有组件的效果
2. **从 Button 组件开始**，这是最简单的，理解基本结构
3. **对照 Storybook 和代码**，看效果是如何实现的
4. **修改代码试试**，比如改改颜色、尺寸，看效果变化
5. **逐步学习更复杂的组件**，理解组件之间的组合使用

```bash

# 构建组件库
npm run build

# 代码检查
npm run lint
```

## 📝 示例：理解 Button 组件

1. **打开 Storybook**：`npm run storybook`，找到 Button 章节
2. **查看代码**：打开 `src/components/Button/button.tsx`
3. **理解结构**：
   - 定义了 Button 的类型（ButtonType, ButtonSize）
   - 使用 `classNames` 库来动态组合 CSS 类名
   - 根据 `btnType` 决定渲染 `<button>` 还是 `<a>` 标签
4. **查看样式**：打开 `src/components/Button/_style.scss` 看样式定义
